= Office-stamper AsciiDoc Preview Extension

The `asciidoc-preview` module is a powerful extension for the Asciidoctor ecosystem.
It allows documentation authors to show exactly how their AsciiDoc templates will appear when rendered as office document comments.

== The `preview::[]` Macro

This block macro can be used in any AsciiDoc document (such as those processed by `maven-site-plugin` or `asciidoctor-maven-plugin`).

=== Syntax

[source,asciidoc]
----
preview::path/to/template.adoc[theme=word, format=png, dpi=300]
----

=== Attributes

[cols="1,3"]
|===
|Attribute |Description

|`theme`
|The editor skin to use (`word`, `gdocs`, `libre`). Defaults to `word`.

|`format`
|The image format (`png` or `svg`). Defaults to `png`.

|`dpi`
|The resolution for PNG export. Defaults to `96`.
|===

== How it Works

1. The extension locates the referenced `.adoc` file.
2. It uses `AsciiDocCompiler` to render the content into the requested format and theme.
3. It saves the resulting image to the project's build directory (`target/generated-previews`).
4. It generates an `<img>` tag (or object tag for SVG) to embed the image in the final HTML.

== Documenting Comments and Placeholders

The preview engine is designed to show how placeholders and comments look in the target editor.

=== Placeholders

Standard AsciiDoc syntax for bold, italic, and source blocks can be used to simulate editor content.
Placeholders like `{{name}}` are rendered as regular text.

=== Comments

To simulate editor comments, use the `comment::[]` block macro.

[source,asciidoc]
----
comment::[id="c1", author="John Doe", value="This is a comment.", start="1,0"]
----

* `id`: A unique identifier for the comment.
* `author`: The name of the comment author.
* `value`: The content of the comment.
* `start`: The position of the block to which the comment is attached (block index, inline offset - currently only block index is used).

The comment will be rendered in a side panel with a dashed connector line to the corresponding content block.

== Example Scene

Here is an example of a complete preview scene:

[source,asciidoc]
----
= Document Title

This is a paragraph with a {{placeholder}}.

comment::[id="c1", author="Jane Smith", value="Please replace this placeholder with the actual name.", start="1,0"]

* First item
* Second item
----
